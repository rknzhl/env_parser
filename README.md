# safe-env-parser

Программа читает `.env`-файл, вычисляет значения переменных (включая подстановки вида `$VAR`) и выводит результат в JSON. Исполнение команд через `$(...)` и неэкранированные бэктики запрещены

## Зачем это нужно

Файлы `.env` задают переменные окружения. Формат простой (`KEY=value`), но поддерживает подстановку переменных:

```
NAME=Ivan
GREETING="Hello, $NAME"    # вычислится как "Hello, Ivan"
```

Проблема: многие запускают `source file.env` напрямую в терминале. Это опасно, потому что файл может содержать произвольный shell-код, например `PASSWORD=$(cat /etc/passwd)`. Наша программа разбирает `.env` безопасно: подстановки раскрыват, исполнение кода запрещает

## Грамматика

Формат `.env` описывается грамматикой в нотации PEG (Parsing Expression Grammar)

Операторы PEG:
- `e1 e2` — конкатенация: сначала e1, затем e2
- `e1 / e2` — упорядоченный выбор: пробуем e1, если не подошло — e2
- `e*` — ноль или больше повторений e
- `e+` — одно или больше повторений e
- `e?` — ноль или одно вхождение e
- `!e` — отрицание: успех, если e не совпало
- `"..."` — точное совпадение строки
- `[a-z]` — любой символ из диапазона
- `ANY` — любой символ

### Файл и строки

```
EnvFile = Line*
Line    = Blanks (Assignment / Comment / "")
Blanks  = (" " / "\t")*
```

Файл состоит из строк. Каждая строка — присваивание, комментарий или пустая. Ведущие пробелы и табы игнорируются

### Присваивание

```
Assignment = ("export" " "+)? Name Blanks "=" Blanks Value
```

Поддерживается необязательный префикс `export`. Пробелы вокруг `=` допустимы

```
NAME=Ivan
export PORT = 3000
DEBUG = "true"
```

### Имя переменной

```
Name = [a-zA-Z_] [a-zA-Z0-9_]*
```

Имя начинается с буквы или `_`, далее могут идти буквы, цифры и `_`. Цифра в начале недопустима

### Значение

```
Value = DoubleQuoted / SingleQuoted / Raw
```

Три вида значений: в двойных кавычках, в одинарных, без кавычек. Проверяются в этом порядке

### Двойные кавычки

```
DoubleQuoted = '"' DblChunk* '"'
DblChunk     = Escape / VarRef / (!'"' !'$' !'\\' !'`' ANY)
```

Внутри двойных кавычек работают подстановки переменных и escape-последовательности. Каждый фрагмент — escape, ссылка на переменную или обычный символ (не `"`, `$`, `\`, бэктик)

```
"Hello, $NAME"           # подстановка $NAME
"Hello, ${FULL_NAME}!"   # подстановка ${FULL_NAME}
"price: \$5"             # буквальный $
"line1\nline2"           # перенос строки
```

### Одинарные кавычки

```
SingleQuoted = "'" SqContent* "'"
SqContent    = !'`' !"$(" !'\'' ANY
```

Внутри одинарных кавычек всё буквально, подстановок нет. Запрещены бэктики и `$(`

```
'Hello $NAME'    # вернёт строку "Hello $NAME"
```

### Без кавычек (raw)

```
Raw      = RawChunk* Comment?
RawChunk = VarRef / (!'$' !'#' !'`' !"$(" ANY)
```

Значение читается до конца строки или до `#`. Подстановки работают, пробелы перед комментарием обрезаются

```
NAME=Ivan
GREETING=Hello $NAME
PORT=3000   # комментарий
```

### Подстановка переменных

```
VarRef = "${" Name "}" / "$" Name
```

Работает в двойных кавычках и в значениях без кавычек. В одинарных кавычках не работает

### Escape-последовательности

```
Escape  = "\\" EscChar
EscChar = "n" / "t" / "r" / "\\" / '"' / "$" / "`"
```

Работают только внутри двойных кавычек:

| Запись | Результат |
|--------|-----------|
| `\n`   | перенос строки |
| `\t`   | табуляция |
| `\r`   | возврат каретки |
| `\\`   | обратный слэш |
| `\"`   | кавычка |
| `\$`   | буквальный знак доллара |
| `` \` `` | буквальный бэктик |

### Комментарии

```
Comment = "#" ANY*
```

Комментарий начинается с `#` и идёт до конца строки

### Запреты

Программа не выполняет содержимое `.env` файла как shell-код. Поэтому следующие конструкции запрещены:

```
X=$(command)
X=`command`
```

## Алгоритмы

### Парсер (`src/Parser.hs`)

Реализует PEG-грамматику методом рекурсивных функций. Каждая функция соответствует одному правилу: `parseEnvFile` для `EnvFile`, `parseDoubleChunks` для `DoubleQuoted` и т.д.

Результат работы парсера — синтаксическое дерево (`EnvFile`)

### Вычислитель (`src/Evaluator.hs`)

Обходит синтаксическое дерево и вычисляет строковые значения переменных. Переменная может использоваться до её объявления в файле

Работает в два прохода:
1. `buildDefs` — строит таблицу `[(имя, AST-значение)]`, проверяет дубликаты
2. `evalVar` — вычисляет переменную рекурсивно, кэширует результат, обнаруживает циклы через путь вычисления

### Вывод (`src/JsonOutput.hs`)

Засовывает список `[(имя, значение)]` в JSON-объект

## Структура проекта

```
src/Ast.hs         типы синтаксического дерева
src/Parser.hs      парсер (.env -> AST)
src/Evaluator.hs   вычислитель (AST -> [(имя, значение)])
src/JsonOutput.hs  сериализация в JSON
app/Main.hs        точка входа
test/Spec.hs       тесты
examples/          примеры .env файлов
```

## Сборка и запуск

```bash
cabal build
cabal run -v0 safe-env-parser -- <файл.env>
cabal test --test-show-details=direct
```

```bash
cabal run -v0 safe-env-parser -- examples/good.env
# {"NAME": "Ivan", "GREETING": "Hello, Ivan", ...}

cabal run -v0 safe-env-parser -- examples/dangerous.env
# Parse error: error: $(...) is forbidden

cabal run -v0 safe-env-parser -- examples/cycle.env
# Error: cycle detected: A -> B -> C
```



## Ошибки

| Сообщеие | Причина | Пример |
|-----------|---------|--------|
| `Parse error: error: $(...) is forbidden` | попытка исполнения команды | `examples/dangerous.env` |
| `Parse error: error: backtick is forbidden` | бэктик-подстановка | `examples/backtick.env`, `examples/backtick_sq.env` |
| `Parse error: error: unclosed double quote` | незакрытая двойная кавычка | `examples/unclosed_quote.env` |
| `Parse error: error: unknown esc \q` | неизвестная escape-последовательность | `examples/bad_escape.env` |
| `Parse error: error: bad variable name near ...` | имя переменной начинается с цифры | `examples/bad_name.env` |
| `Error: unknown variable: $X` | переменная X не объявлена | `examples/unknown.env` |
| `Error: cycle detected: A -> B -> C` | циклическая зависимость | `examples/cycle.env` |
| `Error: duplicate variable: X` | имя X определено дважды | `examples/duplicate.env` |
